Authentication
Every request needs an x-api-key header.

The API key cannot be sent through a browser address bar. Use Postman, curl, or fetch()/JS with a custom header. Use the live tester below to test directly.

GET/api/proxy-video
Streams the actual video bytes. Required for TikTok — see note below.
https://all-media-downloader-api.onrender.com/api/proxy-video?proxy_token=
Query parameter
url — the video link you want to resolve
Full example request
GET https://all-media-downloader-api.onrender.com/api/download?url=https://vt.tiktok.com/XXXXXXX/ Header: x-api-key: m41nul
TikTok note: TikTok's CDN rejects direct fetches of the video_url from anywhere other than this server. So for TikTok, video_url is informational only — to actually get the video bytes, call /api/proxy-video with the proxy_token from the response, e.g.
/api/proxy-video?proxy_token=<value from response>
This streams the already-downloaded file directly (binary video/mp4, not JSON). The token is single-use and expires shortly after resolving, so call it right after /api/download//api/tiktok, not later. Facebook and Instagram don't need this — their video_url can be fetched directly.
